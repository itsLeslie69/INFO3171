Buttons for Data Dashboard

Buttons on the data dashboard should serve clear purposes, enabling users to quickly perform actions. Below are the various button types used in the Star Air dashboard, categorized by function and state.

Priority Buttons

These buttons indicate different levels of action priority, such as primary actions, warnings, and general actions. They help users navigate the dashboard based on task importance.

Disabled Buttons

Disabled buttons are used when an action is not yet available or the user is not authorized to perform it. These buttons have a grayed-out appearance and cannot be interacted with.

Usage

Apply the disabled attribute directly to the button tag to indicate that the button is inactive. Disabled buttons are useful for guiding users on which actions are available.

Loading Buttons

These buttons are displayed when the system is processing a request. They provide feedback to the user, indicating that an action is in progress.

Usage

Toggle the class button-loading dynamically when a request starts, and remove it once the process completes. Use these for actions that may take time, such as data retrieval or submission.

Buttons with Icons

Buttons paired with icons enhance user experience by adding visual cues. These are especially useful for repetitive actions like saving, editing, or deleting data.

Usage

Use icon classes like icon-save, icon-edit, and icon-trash inside the button tag to add icons before the text. Ensure that these icons have enough space and are visually aligned with the button text.

Pagination Buttons

Pagination buttons help users navigate through large sets of data, such as tables or reports. These are typically used for previous and next navigation.

Usage

Wrap multiple pagination buttons inside a button-group container. Use icons like icon-arrow-left and icon-arrow-right to visually distinguish navigation direction.
